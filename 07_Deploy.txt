3. 배포를 위한 절차를 알려줘

1. Vercel 배포 절차
사전 준비
Git 저장소 준비
GitHub/GitLab/Bitbucket에 원격 저장소 생성
프로젝트를 Git으로 관리 중인지 확인
빌드 테스트
   cd web   npm run build
빌드가 성공해야 합니다.
배포 단계
Step 1: Vercel 계정 생성 및 로그인
Vercel 접속
GitHub/GitLab 계정으로 로그인 (권장)
Step 2: 프로젝트 Import
Vercel 대시보드 → "Add New..." → "Project"
Git 저장소 선택
프로젝트 설정:
Framework Preset: Next.js (자동 감지)
Root Directory: web (프로젝트 루트가 web 폴더인 경우)
Build Command: npm run build (기본값)
Output Directory: .next (기본값)
Install Command: npm install (기본값)
Step 3: 환경 변수 설정 (중요)
Vercel 프로젝트 설정 → Environment Variables에서 다음 변수 추가:
# Supabase 설정NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.coNEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-keySUPABASE_SERVICE_ROLE_KEY=your-service-role-key# MQTT 설정 (HiveMQ Cloud)MQTT_BROKER_URL=wss://your-cluster-id.s1.region.hivemq.cloud:8884/mqttMQTT_USERNAME=your-cluster-usernameMQTT_PASSWORD=your-cluster-passwordMQTT_CLIENT_ID=smartfarm-web-client
주의사항:
NEXT_PUBLIC_*는 Production, Preview, Development 모두에 설정
SUPABASE_SERVICE_ROLE_KEY, MQTT_PASSWORD는 Production에만 설정 권장
각 환경별로 다른 값이 필요하면 환경별로 설정
Step 4: 배포 실행
"Deploy" 클릭
Vercel이 자동으로:
npm install 실행
npm run build 실행
배포 진행
배포 완료 후 https://your-project.vercel.app URL 제공
Step 5: 배포 확인
제공된 URL로 접속해 동작 확인
Vercel 대시보드에서 배포 로그 확인
추가 설정 (선택사항)
vercel.json 생성 (필요시)
프로젝트 루트에 vercel.json 생성:
{  "buildCommand": "cd web && npm run build",  "outputDirectory": "web/.next",  "installCommand": "cd web && npm install",  "framework": "nextjs",  "regions": ["icn1"]}
커스텀 도메인 설정
프로젝트 설정 → Domains
원하는 도메인 추가
DNS 설정 안내에 따라 설정
